## 二、项目概述

本项目旨在为小区邻里之间提供便捷的物品借用管理服务。用户可以在平台上登记闲置物品、发布借用需求、查看物品列表，并使用租赁费用计算器计算借用费用。

### 功能模块

1. **首页**: 平台介绍、快捷跳转入口、欢迎语展示
2. **闲置物品列表页**: 查看全部可借物品、新增闲置物品
3. **借用费用计算器**: 输入借用天数和单日租金，自动计算总费用
4. **API接口**: 首页欢迎、物品列表查询、新增物品、费用计算

## 三、技术实现

### 3.1 后端技术

- **框架**: Flask 2.3.3
- **数据库**: SQLite（JSON文件存储）
- **跨域支持**: Flask-CORS 4.0.0
- **API设计**: RESTful API，包含以下接口：
  - `GET /api/index` - 首页欢迎接口
  - `GET /api/goods` - 查询所有闲置物品
  - `POST /api/goods` - 新增闲置物品
  - `GET /api/calc_rent` - 租赁费用计算

### 3.2 前端技术

- **框架**: Next.js 16.1.0
- **语言**: TypeScript
- **路由**: Next.js App Router
- **样式**: Tailwind CSS
- **API调用**: 原生fetch

### 3.3 部署方案

- **平台**: Vercel（Next.js官方推荐）
- **部署方式**: GitHub自动集成，push代码自动触发部署
- **部署地址**: https://flask-next-demo.vercel.app

## 六、遇到的问题及解决方案

### 问题1：Vercel部署失败（rootDirectory配置错误）

**原因**: 最初项目使用 `frontend/` 子目录存放前端代码，Vercel配置中设置了 `rootDirectory: "frontend"`，但vercel.json配置有误导致验证失败。

**解决**: 将前端代码从 `frontend/` 移动到仓库根目录，删除 `vercel.json`，让Vercel自动检测Next.js项目。

### 问题2：国内无法访问Vercel部署地址

**原因**: Vercel的 `.vercel.app` 域名在国内被屏蔽。

**解决**: 这是网络环境问题，不影响项目部署本身。部署状态在Vercel控制台显示正常，可作为实训成果提交。

### 问题3：Git推送网络超时

**原因**: 国内访问GitHub网络不稳定。

**解决**: 使用代理或手机热点进行Git操作，或在开发环境直接推送。
